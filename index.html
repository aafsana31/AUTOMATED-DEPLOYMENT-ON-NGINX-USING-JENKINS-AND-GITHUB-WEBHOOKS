AUTOMATED DEPLOYMENT Using Jenkins, Github Webhooks, and Nginx

Project Overview

This Project demonstrates an automated deployment pipeline that integrates Jenkins for continuous integration, Github Webhooks for triggering builds, and Nginx for reverse proxy management. The purpose of this system is to streamline the deployment process, reducing manual intervention and enhancing productivity in a microservices-based architecture.

Tools Used

  • Jenkins: A popular open-source automation server used for building, testing, and deploying code.
  • Github Webhooks: Used to trigger Jenkins builds whenever changes are made to the repository.
  • Nginx: A high-performance web server and reverse proxy server used to manage traffic and distribute requests to different microservices.
  • Ngrok: used to expose local web servers or other services to the internet, making them accessible from anywhere.

System Architecture

The system architecture consists of the following components:

  • GitHub Repository: The source of the code, where any push or merge triggers the webhook.
  • Jenkins Server: Automatically pulls the latest code, runs the build, and deploys the service to the Nginx server.
  • Nginx: Acts as a reverse proxy, directing traffic to the appropriate microservices running on different ports or Docker containers.

Steps Involved

Step 1: Set up GitHub Webhooks

Configure the GitHub webhook to send a POST request to the Jenkins server whenever a push or merge occurs in the repository.

Step 2: Configure Jenkins

Create a Jenkins pipeline to automate the build, test, and deploy processes. The pipeline will pull the latest code from the repository, build the application, and push it to the Nginx server.

Step 3: Nginx Configuration

Set up Nginx as a reverse proxy to handle requests and forward them to the appropriate microservices running on different ports or Docker containers.
